Introduction

This website documents my COSC427 programming assignment, VGLViewer, which is a 2D vector graphics renderer employing speed-dependent automatic zooming. Vector Graphics Language is a very simple XML-encoded file format for describing 2D geometry and appearance. [SDAZ]

The VGLViewer design has the following features:

  1. Fast rendering using hierarchical bounding boxes.
  2. Minimal coupling to AWT - easy to change to Swing or SWT, for example.
  3. Minimally Java-dependent. VGLViewer can be painlessly ported to C++ (and OpenGL).
  4. Uses integer arithmetic for increased memory efficiency and speed.
  5. Has several reusable packages.

Known Bugs

  1. When run from the command line, VGLViewer will hang after a few tens of seconds of SDAZ zooming. Marquee zoom works OK at the command line however. SDAZ zooming works OK when run from within Together ControlCentre. I suspect a memory leak.
  2. Spaces on either side of commas in VGL vertex lists cause a null pointer exception when loading VGL.
